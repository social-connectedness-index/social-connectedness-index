# SCI Map Maker — web app

A **static, browser-rendered** version of the SCI mapping tool. Anyone can open a
URL and make Social Connectedness Index maps — no server, no R, no login. All map
rendering happens in the visitor's browser with [MapLibre GL JS](https://maplibre.org/),
so the site is just static files on a CDN and scales to viral traffic at zero
marginal cost.

This is **Phase 1**: the lightweight map types (country, state/province (GADM1),
NUTS1, US county, and their region↔country directions), plus comparison maps.
Heavier district-level (GADM2/ADM2) and ZCTA maps are Phase 2 (see the root plan).

## How it works

```
R export pipeline (offline)            Browser (online, static)
  export/export_all.R                    fetch geo/<level>.geojson  (once)
    ├─ geo/<level>.geojson    ─────►     fetch sci/<type>/<id>.json (per source)
    ├─ sci/<type>/<id>.json              compute rel-SCI, breaks, colors (sci.js)
    └─ manifest/groups/palettes          render choropleth + legend (render.js)
                                         export PNG via canvas
```

The browser ships **raw `scaled_sci`** per source and computes the
reference-quantile normalization, legend breaks, and colors client-side — a
faithful port of `src/make_map.R` / `src/mapping_tools.R` (see `src/sci.js`). This
keeps the reference-quantile, custom-breaks, and comparison controls interactive
without re-fetching.

## Build the data (one-time, needs the full local dataset)

From the repo root, with the cleaned shapefiles + SCI data present (run
`./setup.sh` first) and Node + `mapshaper` installed:

```bash
Rscript export/export_all.R            # all Phase-1 levels -> web/public/data/
Rscript export/export_all.R country    # rebuild just one level/type
```

This reads `data/` and writes compact assets to `web/public/data/` (GeoJSON, one
small JSON per source region, and metadata). The 137 GB raw dataset never leaves
your machine — only the ~MB-scale exports are deployed.

## Run locally

```bash
cd web
npm install
npm run dev        # http://localhost:5173
```

## Deploy (free, CDN, viral-scalable)

**Recommended — Cloudflare Pages via direct upload** (keeps the exported data out
of git). One command (builds + uploads):

```bash
cd web
npm run deploy            # = vite build && wrangler pages deploy dist --project-name social-connectedness
```

First run opens a browser to log in to Cloudflare and creates the
`social-connectedness` Pages project. `web/dist` is fully self-contained static
files; the free tier has unlimited bandwidth and a global CDN, so traffic spikes
cost nothing.

### Custom domain (social-connectedness.org)

1. Add the domain to your Cloudflare account (dashboard → Add a site, free plan)
   and update the nameservers at your registrar to the two Cloudflare ones.
2. After `npm run deploy`, go to **Workers & Pages → social-connectedness →
   Custom domains** and add `social-connectedness.org` (and `www`). Cloudflare
   provisions HTTPS automatically.

Alternatives: GitHub Pages or Netlify (point them at `web/dist`). For a
git-connected auto-deploy you must commit `web/public/data/` (currently
gitignored because it can be large).

## Files

- `src/sci.js` — rendering math ported from R (normalize, breaks, palette, labels, comparison)
- `src/render.js` — MapLibre choropleth, legend, fit-to-bounds, PNG export
- `src/main.js` — UI wiring, data loading, state
- `index.html` / `src/style.css` — layout and controls
- `public/data/` — generated by `export/export_all.R` (gitignored)

## Known Phase-1 limitations

- Country filtering is a single preset group (the R app allows multiple + custom).
- NUTS country codes (`EL`, `UK`) differ from ISO-2 (`GR`, `GB`); specific-group
  filtering of NUTS friends is approximate ("All countries" is exact).
- PNG export only (the R app also does PDF/SVG/MP4).
